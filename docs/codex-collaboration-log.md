## Key Product Decisions Made Through Discussion

- Use Voice & Accountability as primary track with Peace & Community as secondary.
- Position Ubuntu Sentinel as community intelligence infrastructure.
- Keep language independent from region.
- Make issue type inferred but human-reviewable.
- Use View for read-only details and Validate for decisions.
- Require validator notes for every decision.
- Make outputs status-aware.
- Keep deterministic fallback honest when OpenAI is not configured.
- Treat resilience zones as a first-class OSF alignment signal.

- Codex explored documents and implementation options.
- The human evaluated strategic fit and product quality.
- Codex implemented slices and debugged failures.
- The human tested flows and caught mismatches.
- Codex revised code and docs until the system matched the intended workflow.
